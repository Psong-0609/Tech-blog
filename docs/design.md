# 디자인 가이드

블로그 MVP의 시각·타이포·레이아웃 규칙은 이 파일을 기준으로 한다. 코드에서는 CSS 변수(`globals.css`) 또는 Emotion theme로 동일 값을 반영한다.

---

## 방향성

- **가독성 우선:** 본문은 긴 글을 부담 없이 읽히게 (줄 길이·행간·대비).
- **단순함:** 장식 최소화, 정보 계층은 크기·여백·색 대비로만 표현.
- **테마:** MVP는 **라이트 단일 테마**로 시작한다. 다크 모드는 MVP 이후 `docs/MVP.md`의 “다음 단계”에서 검토한다.

---

## 디자인 토큰 (초안)

구현 시 변수명은 코드베이스 convention에 맞춰 조정하고, **값 변경 시 이 표를 먼저 수정**한다.

### 색상

| 토큰 의미   | 용도                 | 예시 값 (라이트)                                     |
| ----------- | -------------------- | ---------------------------------------------------- |
| 배경        | 페이지 바디          | `#fafafa` ~ `#ffffff` 범위에서 하나 고정             |
| 표면        | 카드·헤더 구분(선택) | 배경보다 한 단계 어두운 무채색 또는 얇은 보더만 사용 |
| 텍스트 기본 | 본문                 | `#1a1a1a` 근처                                       |
| 텍스트 보조 | 메타·날짜·푸터       | `#666` ~ `#737373`                                   |
| 악센트      | 링크·포커스 링       | 하나만 지정 (예: 블루 또는 브랜드색 1색)             |
| 보더        | 구분선               | `#e5e5e5` 근처                                       |

### 타이포그래피

| 항목           | 권장                                                                                      |
| -------------- | ----------------------------------------------------------------------------------------- |
| 폰트           | 시스템 스택 우선 (`system-ui`, 한글은 OS 기본 고딕). 웹폰트 도입 시 이 표에 패밀리명 추가 |
| 본문 크기      | `16px` ~ `18px` (브레이크포인트에 따라 한 단계만 조정)                                    |
| 본문 행간      | `1.6` ~ `1.75`                                                                            |
| 제목 스케일    | `h1` > `h2` > … 계층이 한눈에 들어오게; 페이지당 큰 `h1`은 하나                           |
| 목록·카드 제목 | 본문보다 굵게, 줄 수 제한 시 `-webkit-line-clamp` 등으로 통일                             |

### 간격 & 레이아웃

| 항목              | 권장                                                                   |
| ----------------- | ---------------------------------------------------------------------- |
| 페이지 좌우 패딩  | 모바일 `16px`~`20px`, 넓은 화면에서 최대 컨테이너 안으로만 증가        |
| 섹션 간 세로 간격 | `24px`~`48px` 단계로 제한된 스케일 사용 (임의 픽셀 난립 금지)          |
| 본문 최대 폭      | 약 `65ch` ~ `72ch` (코드 블록은 가로 스크롤 허용)                      |
| 전역 최대 너비    | 헤더·목록·푸터 정렬용 `max-width` 하나 고정 (예: `720px` 또는 `960px`) |

### 모양

| 항목   | 권장                                                                                |
| ------ | ----------------------------------------------------------------------------------- |
| 코너   | 카드·버튼 등에만 소량의 `border-radius` (예: `6px`~`10px`), 또는 모두 직각으로 통일 |
| 포커스 | 키보드 사용자용 `:focus-visible` 아웃라인 또는 링을 악센트색으로 통일               |

## 컴포넌트별 요구

### Header

- 블로그 이름(로고 텍스트) + 필요 시 단일 네비 링크.
- 높이·패딩은 토큰 간격 스케일만 사용.

### Footer

- 저작 표시·선택 링크; 텍스트는 보조색.

### Container

- 좌우 패딩 + `max-width` 중앙 정렬; 모든 페이지 본문의 공통 래퍼.

### PostCard (목록)

- 제목, 요약(1~2줄), 날짜. 호버 시 링크 피드백(밑줄·배경 미세 변화 중 하나로 통일).

### Prose / Markdown 본문

- `p`, `h2`~`h6`, `ul`/`ol`, `blockquote`, `pre`/`code`, `a`, `img`에 대한 여백·크기 규칙을 한 컴포넌트·클래스에 모은다.
- 인라인 코드와 코드 블록 배경·폰트 구분을 명확히 한다.
