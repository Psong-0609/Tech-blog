# Cursor — Tech-blog 프로젝트 진입점

Chat/Composer에서 작업할 때 **먼저 `agent.md`**를 참고합니다. 여기서는 Cursor 사용자를 위한 요약과 저장소 관습만 적습니다.

## 프로젝트 한 줄 요약

Next.js(App Router) 기반 기술 블로그 — 글은 저장소 루트 **`content/*.md`**(**`src/` 밖**, 형제 디렉터리), 편집은 **TipTap**·저장은 Markdown, 글 CRUD·카테고리·검색·다크모드 및 성능·**현재 `src/app` 중심 폴더 구조**·**하위 레이어 순수 함수 지향**·접근성 목표는 `agent.md`에 정의되어 있습니다. **보안·가용성(모니터링·알림 등)은 추후** 다룹니다.

## 빠른 체크리스트 (변경 후)

| 영역 | 확인                                                                                                      |
| ---- | --------------------------------------------------------------------------------------------------------- |
| 기능 | 작성/수정/삭제(루트 `content/*.md`, `src/` 외부, TipTap), 카테고리, 검색(카테고리+키워드), 테마 전환 동작 |
| 성능 | 불필요한 클라이언트 코드, 큰 이미지, 레이아웃 시프트                                                      |

## 로컬 명령

```bash
pnpm dev    # 개발 서버
pnpm build  # 프로덕션 빌드
pnpm lint   # ESLint
```

## 파일 역할

- **`agent.md`**: 에이전트·기여자 공통의 요구사항·아키텍처 원칙·접근성 상세.
- **`cursor.md`**: Cursor 작업 시 열어볼 요약본(본 파일).

세부 규칙을 `.cursor/rules` 등으로 추가할 경우, `agent.md`와 충돌하지 않게 한 줄로 상호 참조를 적습니다.
